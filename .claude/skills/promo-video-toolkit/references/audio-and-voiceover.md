# 音声：AIナレーション・BGM/SE・字幕

動画の音声まわり。**無音・自動再生が前提**（LP/SNS）なので字幕が主役、音声は補助。explainer/デモではナレーションが効く。

## 0. 前提：無音で成立させ、音声は"上乗せ"

- LPヒーロー・SNS広告は**ミュート自動再生**。テロップで意味が通ること（`video-strategy.md`）
- explainer・YouTube・営業資料は音声ありで没入感
- **どの動画も字幕（キャプション）を付ける**：A11y・無音視聴・SEO/AI検索（トランスクリプト）に効く

## 1. AIナレーション（音声合成）

| サービス | 特徴 | コスト感 |
|---------|------|---------|
| **ElevenLabs** | 自然さ・多言語・日本語良好・感情表現 | 文字数/分課金。安価〜中。商用は要該当プラン |
| **OpenAI TTS** | API で手軽・複数ボイス | API 従量・安価 |
| **Google / Azure TTS** | 安定・SSML 細制御・無料枠あり | 無料枠＋従量 |
| **にゃんTTS等の日本語特化/OSS** | コストを抑えたい時 | 品質要確認 |

- **日本語の品質チェック**：固有名詞・英数字混在・アクセント・間。読み間違いは SSML（`<phoneme>`/`<break>`）や表記調整で補正
- **台本は話し言葉**で（`video-strategy.md` の台本原則）。句読点で間を作る
- **商用利用とボイスの権利**：生成音声の商用可否・声の権利をプランで確認。実在人物の声を模さない
- **AI音声の開示**：媒体・広告審査により必要な場合あり

```
SSML 例：<speak>シフト作成が、<break time="300ms"/>3時間から10分に。</speak>
```

> **関連スキル**：日本語TTSの**発音調整・SSML・プロバイダ選定**をさらに深掘りしたい場合は `ai-calling-japan` Skill の音声合成節（`references/service-landscape.md` の TTS、`cost-and-accuracy-jp.md`）も参考になる。
> **ただし通話向けの議論（低遅延ストリーミング・電話帯域 8kHz・barge-in 割り込み停止・通話録音の同意/越境移転）は動画ナレーションには不要**。動画は**オフライン・高音質（48kHz）前提**で、発音/SSML/声選び/商用ライセンスの部分だけを読み替えて使うこと。

- **ライセンスが命**：商用利用可・帰属表示要否・再配布条件を必ず確認
- 入手先（要規約確認）：
  - ロイヤリティフリー音源サービス（有料サブスクで商用可が明確なもの）
  - YouTube オーディオライブラリ（商用可・帰属要否は曲ごと確認）
  - フリー音源（DOVA-SYNDROME / 甘茶の音楽工房 等、各サイトの規約に従う）
- **トーン**を design-system/visual-style-guide に合わせる（信頼系SaaSに過剰なEDMは×）
- SE は控えめに（UI操作の pop、登場の whoosh 程度）

## 3. 字幕・キャプション

2種類を用途で使い分ける：

| 種類 | 形 | 用途 |
|------|----|----|
| **焼き込みテロップ**（オープンキャプション） | 映像に固定描画（Remotion） | LP/SNS。装飾と一体・必ず見える |
| **クローズドキャプション**（.srt/.vtt） | 別ファイルで on/off | YouTube・`<track>`・A11y・SEO |

- Remotion なら台本テキストをタイムライン化してテロップを生成（フォント=ブランド）
- 文字起こしは Whisper 等で自動生成 → 校正して .srt/.vtt 化
- 可読性：十分なコントラスト・縁取り/下地・1–2行・表示は最低 1.5秒
- SNSは安全マージン内（UI被り回避）、中央寄せ・特大

## 4. 音量・書き出し（ラウドネス）

- **ラウドネス基準**に合わせる（配信先で-14 LUFS 前後が一般的、SNSは各社で正規化される）
- ナレーションを聞き取りやすく、BGMはダッキング（ナレーション時に自動で下げる）
- クリッピング（音割れ）回避。ピークに余裕（-1 dBTP 目安）
- 書き出しは AAC・適切なビットレート（`delivery-and-distribution.md`）
