# 動画戦略：タイプ別の設計・構成テンプレ・絵コンテ

「誰に・どの場面で・何を見せ・どう動いてもらうか」を決める。手法選び（Remotion/録画/AI）の前に、**動画タイプごとの目的・尺・比率・構成**を確定する。

|--------|---------|------|----|----|--------|------|
| **LPヒーロー動画** | 営業サイトのトップ | 一目で価値を伝え離脱を防ぐ | 6〜15秒（ループ） | 16:9 / 横長背景 | Remotion ＋ AIのB-roll差し色 | 無音ループ |

## 1. 共通設計原則

- **無音・自動再生前提**：音声なしで意味が通る（テロップ必須）。`audio-and-voiceover.md`
- **冒頭3秒フック**：何のSaaSで何が良くなるか／意外な事実／課題の一言
- **1動画1メッセージ**：詰め込まない。CV（無料トライアル等）に向けて1本道
- **ブランド整合**：色・フォント・ロゴモーションは design-system / visual-style-guide
- **比率は最初から目的別に**：16:9 と 9:16 は撮り直しでなく**別構図で設計**（後変換は崩れる）
- **CTA**：終盤にロゴ＋一言＋URL/ボタン誘導

## 2. タイプ別の構成テンプレ

### LPヒーロー動画（6〜15秒・無音ループ）
```
0–3s  フック：プロダクトの一番の価値を1カット（実画面 or 象徴ビジュアル）
3–10s 価値の連続提示：主要ベネフィット2–3点をテロップ＋動き
10–15s ロゴ＋タグライン（ループでフックに戻る）
```
※ LPでは動画は「装飾」ではなく**最初のスクロール前で価値を伝える主役**。ただしLCPを壊さない（`delivery-and-distribution.md`）。

### プロダクトデモ動画（60〜120秒）
```
0–8s   課題の提示＋「これで解決」宣言
8–90s  実操作（録画）を3–5ステップ：ログイン→主要機能→成果が出る瞬間
       各ステップにテロップで「何をしているか」「何が嬉しいか」
末尾    Before/After or 成果数値 → CTA（無料で試す）
```
※ 操作は**速度1.2–1.5倍＋不要待ち時間カット**。マウス強調・ズームで注目誘導。

### SNS縦型広告（15〜30秒・9:16）
```
0–2s  強フック：問い・数字・意外な映像（スクロールを止める）
2–20s 課題→解決を超高速で（テロップ大きく・1カット1–2秒）
20–30s ブランド＋CTA（プロフィールリンク/検索誘導）
```
※ 上下の安全マージン（UI被り回避）。テロップは中央寄せ・特大。

### explainer/説明動画（60〜90秒）
```
0–10s  共感（あるあるの課題）
10–25s 原因／なぜ大変か
25–60s 解決＝プロダクトの仕組みをモーショングラフィックスで
60–90s 導入効果＋CTA
```
※ ナレーション台本→絵コンテ→Remotion 実装の順。

## 3. 絵コンテ（ストーリーボード）の作り方

実装前に**カット単位**で書き出す。Markdown 表で十分：

```
| # | 尺 | 画 | テロップ | 音声/SE | 手法 |
|---|----|----|---------|---------|------|
| 1 | 0–3s | ダッシュボードがスッと表示 | 「シフト作成、3時間が10分に」 | BGM in | 画面録画 |
| 2 | 3–6s | AIが自動でシフトを埋める | 「希望と店舗ルールを両立」 | — | Remotion |
| 3 | 6–10s | スタッフがスマホで確認 | 「共有もワンタップ」 | SE: pop | AI B-roll |
| 4 | 10–15s | ロゴ＋CTA | 「14日間無料」 | BGM out | Remotion |
```

## 4. 台本（ナレーション/テロップ原稿）

- 1カット = 1メッセージ、1文は短く（テロップは12〜20字目安）
- 数字・固有名詞で具体化（「多くの店舗」→「導入店舗の82%」）
- 抽象語（革新的・次世代）を避ける（営業サイトのコピー方針と同じ）
- ナレーションは話し言葉、句読点で間を作る

## 5. 量産設計（野口さんの肝）

1案件で作って終わりにせず、**Remotion でテンプレ化**して次案件に流用：
- ブランドトークン（色/フォント/ロゴ）を props で差し替え
- 「ヒーロー」「機能3点」「料金」「CTA」をコンポーネント化
- 案件ごとに変わるのは文言・色・画面キャプチャだけ、という状態にする
